# Purpose

The main purpose of this project is to write a high responsive healthcheck that simulates a real life scenario.

## Disclaimer

No AI is used in order to prove knowledge and experice real issues using different strategies.

## Scenarios

This should simulate the following scenarios:
- No parallelism: Paying off the price of all sequential calls
- Parallelism without timeout: Paying the most expensive call time
- Parallelistm with timout: using timeout budget and see how many calls successfully returned
